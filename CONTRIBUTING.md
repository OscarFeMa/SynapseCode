# Contributing / Contribuir

## 🇪🇸 Español

Gracias por tu interés en SynapseCode. Para contribuir:

1. **Issues**: Revisa si ya existe un issue abierto antes de crear uno nuevo.
2. **Fork + Branch**: Trabaja en una rama descriptiva (`feature/nueva-funcionalidad`, `fix/error-x`).
3. **Commits**: Usa mensajes claros en inglés o español, preferiblemente con prefijo (`feat:`, `fix:`, `refactor:`, `docs:`).
4. **Tests**: Asegura que `python -m pytest backend/tests/ -v` pase antes del PR.
5. **Linting**: Ejecuta `ruff check backend/` y corrige cualquier advertencia.
6. **PR**: Describe qué cambia y por qué. Incluye capturas si aplica.

## 🇬🇧 English

Thank you for your interest in SynapseCode. To contribute:

1. **Issues**: Check for existing open issues before creating a new one.
2. **Fork + Branch**: Work on a descriptive branch (`feature/new-feature`, `fix/bug-x`).
3. **Commits**: Write clear messages in English or Spanish, preferably with a prefix (`feat:`, `fix:`, `refactor:`, `docs:`).
4. **Tests**: Ensure `python -m pytest backend/tests/ -v` passes before submitting a PR.
5. **Linting**: Run `ruff check backend/` and fix any warnings.
6. **PR**: Describe what changes and why. Include screenshots if applicable.
